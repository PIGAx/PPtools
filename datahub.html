建議使用一個獨立的私人 Repo 專門存放資料，與工具程式碼分開。
第一次設定教學（建立私人 Repo 與 PAT）
  1. 到 GitHub 建立新 Repo：名稱 pp-data，勾選 Private，勾選「Add a README file」（讓 main 分支存在）。
  2. 進 GitHub 右上角頭像 → Settings → 最下方 Developer settings → Personal access tokens → Fine-grained tokens → Generate new token。
  3. Token name 隨意（如 pptools-sync），Expiration 建議選 90 天或自訂一年。
  4. Repository access 選「Only select repositories」→ 只勾 pp-data。
  5. Permissions → Repository permissions → Contents 設為 Read and write，其他都不用動。
  6. 產生後複製 github_pat_ 開頭的字串，貼到上方 PAT 欄位，按「儲存設定」再「測試連線」。

安全性說明：PAT 只儲存在這台裝置的瀏覽器 localStorage，網站訪客看不到。因為 PAT 權限只限 pp-data 這個 Repo 的內容讀寫，即使外洩影響範圍也僅止於此。PAT 到期後重新產生一個貼上即可。
